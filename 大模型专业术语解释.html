    ✅大模型专业术语解释

    1.大模型基础

    LLM

    大语言模型（Large Language Model） ——是一类基于深度学习的神经网络模型，具备生成自然语言文本的能力。 LLM通过对大量文本数据进行预训练，学习语言中的词汇、语法、语义规律。它们在自然语言处理任务中表现优异，如对话生成、文本摘要、翻译等。代表模型包括 GPT 系列、Claude、Gemini 等。

    Transformer

    ——是一种以注意力机制为核心的神经网络架构，是现代大模型的基础结构。 由Google在2017年提出的“Attention is All You Need”论文中首次提出。其核心优势是并行计算能力强，能有效处理长文本序列，是GPT、BERT等大模型的底层结构。

    Prompts

    提示词工程（Prompt Engineering） ——是设计输入文本以引导模型输出预期结果的技术。 提示词可以是问题、指令或范例，通过精心设计提示语（Prompt）来控制模型的行为，提高输出的相关性和准确性。例如，“请以律师口吻撰写以下合同内容”。

    Tokens

    ——是模型在处理语言数据时的最小计算单位。 一个Token可以是一个字符、词干或子词，具体由分词器（Tokenizer）定义。模型的输入输出都以Token为基本单位，Token数量直接影响上下文窗口长度与计算资源消耗。

    Context Window

    上下文窗口 ——是指模型在一次生成中可以“记住”的Token数量范围。 详细内容：不同模型有不同的上下文窗口大小（如GPT-4为128k Tokens）。窗口过小时模型容易“遗忘”前文信息；窗口增大可提升推理质量，但计算开销也随之增加。

    Parameters

    参数量 ——表示模型中需要训练和优化的权重数量，直接体现模型的复杂度和能力。 详细内容：如GPT-3拥有1750亿参数，GPT-4据传有超过万亿参数。参数越多，模型理论上越强大，但也带来更高的训练与推理成本。

    2.热门技术

    近年来，为提升大模型实用性，许多新技术被广泛采用，如函数调用、Agent 架构等。这些技术不断拓展 LLM 的能力边界。

    Function Calling

    函数调用 指大模型通过结构化方式调用外部函数或API以执行实际操作的能力。 例如，用户输入“今天北京天气如何？”，模型不直接生成天气信息，而是识别意图并调用天气API返回结果。这种方式提升了模型的可控性和准确性。

    MCP

    模型上下文协议（Model Context Protocol） MCP是一种用于描述模型调用链中上下文的协议标准。 它定义了如何将多个函数调用、API结果、模型思维过程统一组织到上下文中，让多步推理和Agent协作更加清晰、高效。

    RAG

    检索增强生成（Retrieval-Augmented Generation） RAG是一种结合文档检索系统与生成模型的方法，用于提升事实性和上下文覆盖率。 模型先从外部知识库中检索相关信息，再将结果注入生成上下文。常用于企业知识问答、法律法规咨询等场景，显著降低幻觉问题。

    Agent

    智能体 指具备感知、思考、决策与执行能力的自治AI系统。 与传统聊天模型不同，AI Agent可以主动规划任务、调用工具、持久记忆等。典型应用如AutoGPT、OpenDevin、LangGraph等。

    Workflow

    工作流 工作流是将多个模型步骤或函数组合为可复用的处理流程。 在Agent系统中，工作流允许定义任务序列，如“搜索 → 抽取信息 → 调用API → 整合回答”，支持多步骤推理和任务分解。

    3.训练与微调

    本节介绍大模型从训练到定制的全过程，包括预训练、微调、强化学习等方法。理解这些术语有助于把握模型能力如何演进。

    Pre-training

    预训练 是指在海量通用文本上训练模型，以获得语言理解与生成能力的过程。 通常采用无监督学习方式，如预测下一个Token，耗时长但是模型“通才”能力的基础。

    Fine-tuning

    微调 在预训练模型基础上，使用特定领域数据对模型进行再训练。 微调可以使模型在某一任务或行业表现更好，如法律问答、医学诊断等。

    SFT

    监督微调（Supervised Fine-Tuning） 使用人工标注的高质量输入输出对进行微调。 是对基础模型进行任务对齐的重要方式，例如：用户输入的问题和预期正确回答。

    LoRA

    低秩适配（Low-Rank Adaptation） 通过添加少量可训练参数矩阵，对模型进行高效微调。 仅更新部分参数，大幅减少显存占用，常用于边缘设备部署。

    QLoRA

    量化低秩适配（Quantized LoRA） 结合参数量化与LoRA进行微调的优化方案。 详细内容：显著降低模型运行资源需求的同时保持性能，适合低成本部署。

    Distillation

    模型蒸馏 使用大模型（Teacher）的输出作为小模型（Student）的训练目标。 详细内容：保留性能同时压缩模型，是提高推理速度和效率的重要手段。

    Quantization

    模型量化 将模型参数从浮点数压缩为更低精度（如8位整数）以减少资源占用。 降低显存消耗和推理延迟，但需权衡精度损失。

    RLHF

    来自人类反馈的强化学习（Reinforcement Learning from Human Feedback） 模型通过人类对输出结果的好坏评分进行策略优化。 如ChatGPT训练中的“偏好优化”阶段，提高回答质量与人类期望一致性。

    RLAIF

    中文名称：来自AI反馈的强化学习（Reinforcement Learning from AI Feedback） 用AI模型替代人工提供反馈，从而加速训练过程。 提高训练自动化程度，但仍在探索可靠性问题。

    PEFT

    参数高效微调（Parameter-Efficient Fine-Tuning） 总称，涵盖如LoRA等低资源微调方法。 详细内容：核心思想是只更新少量参数，适合资源受限场景或快速定制应用。

    4.大模型架构

    当前主流的大语言模型在架构上各有特色，了解这些架构有助于比较模型能力、适用场景和优化手段。

    GPT

    生成式预训练变换器（Generative Pre-trained Transformer） 是OpenAI提出的基于Transformer架构的语言生成模型系列。 采用Decoder-only结构，擅长生成文本，是当前LLM的代表之一。

    BERT

    双向编码器表示模型（Bidirectional Encoder Representations from Transformers） Google提出的一种只用于理解（非生成）的Transformer模型。 详细内容：采用双向Encoder结构，擅长问答、分类等理解任务。

    MoE

    专家混合架构（Mixture of Experts） 将模型拆分为多个“专家”子模型，每次只激活部分专家。 详细内容：在保持参数量高的同时降低实际推理成本，代表如Google Switch Transformer。

    5.其他知识

    本节包含部分与大语言模型密切相关但跨领域的概念，帮助理解模型行为与性能调节机制。

    CoT - Chain-of-Thought Prompting

    中文名称：链式思维提示 通过引导模型一步步推理提升其复杂任务表现。 详细内容：提示词中包含分步解释，使模型学习类似人类的推理过程，如数学题解决过程。

    Hallucination

    中文名称：幻觉现象 模型生成事实不准确或虚构内容的现象。 详细内容：由于语言模型并不理解“事实”，仅预测最可能的Token序列，因此常会“编造”信息。

    Multimodal

    中文名称：多模态模型 同时处理多种输入模态（文本、图像、语音等）的模型。 详细内容：如GPT-4V、Gemini等可接受图像+文本输入，实现更丰富的人机交互能力。

    Embedding

    中文名称：向量嵌入 将文本转换为向量的表示方法，用于语义匹配、相似度计算等任务。 详细内容：在RAG、搜索召回等场景中广泛应用，可用于构建知识库索引。

    Temperature

    中文名称：温度参数 控制模型输出随机性的参数。 详细内容：值越高（如0.9）结果越多样，值越低（如0.2）结果越稳定。适用于创意 vs 精准场景调节。

    Attention Mechanism

    中文名称：注意力机制 模型用以捕捉输入中重要信息的机制，是Transformer的核心。 详细内容：通过赋予不同词不同权重，使模型能聚焦于更有意义的上下文部分。
